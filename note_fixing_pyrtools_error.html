Fixing rpytools Error

     

  • Occasionally, when I use the ‘reticulate’ package in RStudio, it doesn’t work. I’ll be executing Python chunks like it’s nobody’s business, then all of the sudden, I’ll get an error and my Python chunks that ran great previously will stop working.

  • Occasionally, it will be the ‘r setup’ code chunk that caches code chunks globally that needs to be disabled:

\```{r setup, include=TRUE}
knitr::opts_chunk$set(cache=TRUE)
\```

  • Occasionally, the Python environment isn’t setup properly and I need to refer to these web pages then pray: https://rstudio.github.io/reticulate/articles/python_packages.html https://rstudio.github.io/reticulate/articles/versions.html

  • Occasionally, I get the error: Evaluation error: ModuleNotFoundError: No module named ‘rpytools’

    Well I fixed this one! I needed to copy the rpytools folder originally in this location to this location. I almost pooped my pants when it worked. Also, I omitted my user name in my hyperlinks. If I never have to run Python in R again, it will be too soon.
